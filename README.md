# thesis-data-lab

卒論準備中の学部4年生向けに、Python によるデータ分析・計量経済学の基礎作法を短時間で確認するための教材リポジトリです。

この教材では、各手法を深く学ぶことよりも、卒論で最低限必要になる分析の進め方を身につけることを重視します。

## このリポジトリの考え方

- `notes/` は先生用の講義ノートです。Quarto で管理します。
- `scripts/` は学生向けの実行用コードです。Jupyter 必須にせず、`uv` + Python スクリプトを前提にします。
- `data/` には加工済みデータやデータの出所メモを置きます。
- 生データは必ずしもこのリポジトリに置きません。制約付きデータは repo 外に配置する前提を認めます。
- `output/` には図表の出力を置きます。

## 最小ディレクトリ構成

- `notes/`
- `scripts/`
- `data/processed/`
- `data/metadata/`
- `output/figures/`
- `assets/`

## ノートとコードの役割分担

### `notes/`

- 授業で話す順序を管理する
- 分析の考え方、注意点、図表の見方を書く
- 「授業で話すこと」と「自習に回すこと」を分けて書く
- 実行コードは貼りすぎず、必要最小限の断片にとどめる

### `scripts/`

- 学生がそのまま実行して流れを確認できるコードを置く
- 元データから分析用データを作る処理を残す
- 記述統計、散布図、回帰の順に処理を書く
- 共通のパス処理や補助関数は `scripts/common/` に寄せる

## データ管理の方針

- データの取得元は `data/metadata/` に記録する
- 生データを GitHub に置けない場合を想定する
- 外部 AI に相談するときは、生データを送らない
- 必要なら疑似データをローカルで生成して相談する

## まず書き始める対象

- 第1回: 卒論の分析作法と再現可能な作業フロー
- 第2回: データ確認、記述統計、散布図、単回帰への入口

## 使い始め方

1. Quarto を使って `notes/` のノートを編集する
2. `scripts/session_01/` や `scripts/session_02/` に実行例を追加する
3. 加工済みデータとメタデータを `data/` に置く
4. 図表は `output/figures/` に保存する

今は「書き始められる最小構成」を優先してあります。各回のノートは今後少しずつ増やしていく前提です。
