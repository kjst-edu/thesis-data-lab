第2回 記述統計・散布図・単回帰への入口

スクリプト

1 この回のねらい

いきなり回帰分析に入らず、データの全体像をつかむ順序を確認します。

  1. 変数の意味と単位を確認する
  2. 欠損値の出方を確認する
  3. 記述統計量を出す
  4. 図でデータの形を見る
  5. 予想される符号やメカニズムを言葉で書き出す
  6. 単回帰を実行し、図と事前の見立てに整合するかを確認する

各ライブラリの細かい使い方は公式ドキュメントに譲ります。卒論では、自分が選んだ1つを使い込むほうが効率がよいです。

2 使うデータ

データセット

第1回で使った Penn World Table 11.0（PWT）を再利用し、ここでは 2023 年のクロスセクションを抽出して使います。第1回と同じく、生データ pwt110.dta は .env の RAW_DATA_DIR 配下に置かれている前提です。

重要データの引用条件を確認する

データの配布元には引用条件が示されていることが多くあります。PWT の場合、配布ページに次の指示があります。

Attribution requirement - When using these data (for whatever purpose), please make the following reference:

Feenstra, Robert C., Robert Inklaar and Marcel P. Timmer (2015), “The Next Generation of the Penn World Table” American Economic Review, 105(10), 3150-3182, available for download at www.ggdc.net/pwt

卒論で使うデータごとに「How to cite」「Terms of Use」「License」などの記載を探し、本文や参考文献に反映させます。

スクリプトの取得

スクリプト一式は配布リポジトリの scripts/session_02/ に置いてあります。最新版を取り込むには、自分のプロジェクトのルートで次を実行します。

  • Mac
  • Windows
gh repo clone kjst-edu/thesis-data-lab _tdl -- --depth 1 && cp -r _tdl/scripts/session_02 scripts/ && rm -rf _tdl
gh repo clone kjst-edu/thesis-data-lab _tdl -- --depth 1; Copy-Item _tdl\scripts\session_02 scripts\ -Recurse; Remove-Item _tdl -Recurse -Force

加工データの作成

加工コードは scripts/session_02/01-prepare_pwt_2023.py に分けてあります。中身は次のとおりです。

# scripts/session_02/01-prepare_pwt_2023.py
#
# Penn World Table (PWT 11.0) から 2023 年のクロスセクションを抽出し、
# data/pwt_2023.csv として保存する。欠損値の処理や派生変数の作成は
# 行わず、後段の分析スクリプトに任せる。

from pathlib import Path
import os

import pandas as pd
from dotenv import load_dotenv

load_dotenv()

RAW_DATA_DIR = Path(os.environ["RAW_DATA_DIR"])

YEAR = 2023
OUTPUT_DATA = Path("data/pwt_2023.csv")

USE_COLS = [
    "countrycode", "country", "year",
    "rgdpna", "pop", "emp", "avh",
    "rnna", "hc", "labsh",
]

raw = pd.read_stata(RAW_DATA_DIR / "pwt110.dta", columns=USE_COLS)
df = raw[raw["year"] == YEAR].copy()

OUTPUT_DATA.parent.mkdir(exist_ok=True)
df.to_csv(OUTPUT_DATA, index=False)

実行します。

uv run scripts/session_02/01-prepare_pwt_2023.py

これで data/pwt_2023.csv ができます。以降のチャンクはここから読み込みます。

3 分析ステップ

[1] 変数の意味を確認する

データを読み込んだ直後に、各列が何を表しているかを言葉で説明できる状態にします。

  • 変数名だけで判断しない（income が世帯か個人か、税前か税後か、月か年かは名前ではわからない）
  • 単位を確認する（金額の通貨、人数か割合か、対数を取った後の値か）
  • 集計単位を確認する（個人・世帯・企業・地域・年など）
  • 目的変数と説明変数の候補を、データを見るより先に言葉で挙げる

PWT の場合、変数の単位は配布ページの User Guide に書かれています。たとえば rgdpna は「100万 2017 USD」、pop・emp は「100万人」なので rgdpna / emp はそのまま「労働者1人あたり 2017 USD」になります。この単位確認を省くと、図の縦軸の桁が3桁ずれるといったミスが起きます。

[2] 欠損値を確認する

記述統計に進む前に、各列の欠損が何件あるかを確認しておきます。DataFrame.isna().sum() で列ごとの NaN 件数が出せます。

PWT 2023 年は 185 か国分の行がありますが、変数によっては欠損が多めです。

  • avh（年間平均労働時間）と labsh（労働分配率）は欠損が多く、3割前後の国で取れない
  • hc（人的資本指数）は2割ほど欠損
  • rgdpna、pop、emp、rnna は比較的そろっている

欠損があるときの対処は何通りかありますが、どの選択肢にも独自の偏りが伴います。

  • 行を落とす — 最も単純。ただし PWT の場合、欠損は統計行政が弱い国・規模が小さい国・透明性が低い国に集中するので、残るサンプルが「規模が大きく統計が整った経済」に寄る偏りが入る
  • 対象年を変える — 2022年や2021年に切り替えると、最新値が間に合っていない国を拾えることがある。ただし「2023 年について論じたい」場合は別の話になる
  • 国ごとに利用可能な最新年を取る — サンプル数を保てるが、基準年が国でばらつく。構造的にゆっくり動く変数（人的資本など）に向く
  • 他のデータソースで補う — World Bank WDI、OECD、IMF などで埋める方法もある。出所ごとに定義や対象年が微妙に違うので、本文で出所と定義のズレを明示する

どの選択肢を選んでも、サンプルは何らかの形で偏ります。卒論では「なぜその処理を選んだか」「結果として残ったのはどんな国か」を本文に書き残しておくと、後の議論や反例の説明がしやすくなります。

今回は教材なので、最も単純な「行を落とす」を採ります。rgdpna、emp、rnna、hc のいずれかに欠損がある国を分析サンプルから外します（hc は後段のペアプロットで使うので一緒に落としておきます）。

[3] 分析用の変数を作る

欠損行を落として、対数と労働者あたり指標を計算します。

df = df.dropna(subset=["rgdpna", "emp", "rnna", "hc"]).copy()
df["gdp_per_worker"] = df["rgdpna"] / df["emp"]
df["capital_per_worker"] = df["rnna"] / df["emp"]

[5] 先に図を描く

推定の前に、変数の分布と変数間の関係を目で確認します。同じ図でもライブラリにより向き不向きがあります。卒論ではどれか1つに絞ってかまいません。

以下では seaborn を使って、最低限見ておきたい図を描きます。

ヒストグラム

各変数の分布の形（左右非対称、二峰性、極端な裾）を見ます。

fig, ax = plt.subplots(figsize=(6, 4))
sns.histplot(data=df, x="gdp_per_worker", bins=20, ax=ax)
plt.show()

対数軸でも見ます。

fig, ax = plt.subplots(figsize=(6, 4))
sns.histplot(data=df, x="gdp_per_worker", bins=20, log_scale=True, ax=ax)
plt.show()
fig, ax = plt.subplots(figsize=(6, 4))
sns.histplot(data=df, x="hc", bins=20, ax=ax)
plt.show()

散布図

2変数の関係の向き、外れ値、直線で近似してよさそうかを見ます。

fig, ax = plt.subplots(figsize=(6, 4))
sns.scatterplot(data=df, x="capital_per_worker", y="gdp_per_worker", alpha=0.7, ax=ax)

散布図 + 回帰直線

sns.regplot を使うと、散布図に最小二乗の回帰直線を重ねられます。推定する前に「直線で近似してよさそうか」を視覚的に確認できます。

fig, ax = plt.subplots(figsize=(6, 4))
sns.regplot(
    x=np.log(df["capital_per_worker"]),
    y=np.log(df["gdp_per_worker"]),
    line_kws={"color": "red"},
    scatter_kws={"alpha": 0.6},
    ax=ax,
)
ax.set_xlabel("log(capital per worker)")
ax.set_ylabel("log(GDP per worker)")
plt.show()

ヒント

散布図で外れ値や非線形な広がり方が見えたら、対数変換やサンプル制限を検討します。

3.0.1 外れ値を見つけたら

外れ値らしき点が見えたら、まず異常値か実態かを判断します。配布元のドキュメントに当たり、入力ミスや単位違いの疑いがあれば修正、実態であれば原則そのまま残します。

それでも除外を選ぶ場合は、次の形で作業記録を残します。

  • 除外条件をコードで書く（df.query(...) や df[df['x'] < ...] の形でスクリプトに残す）
  • 除外件数を print で確認する
  • 可能なら、除外あり / なしの両方で推定を回し、結果がどう変わるかを比較する

コードに条件が残っていれば、Git の履歴から条件と日付が遡れます。卒論本文には、除外条件と件数を1〜2行で書き添えます。

[6] 回帰の前に考えること

図を見た段階で、推定を回す前に次のことを言葉で書き出します。

  • 予想される符号は何か
  • その理由は何か（経済理論・先行研究・常識）
  • 逆の符号が出るとすれば、どのような要因が考えられるか（逆因果、欠落変数、測定誤差）

今回の場合は、Solow 型の生産関数を念頭に「資本/労働比率が高いほど労働生産性が高くなるはず」と予想できます。傾きは正、対数同士の回帰なので、傾き = 資本に関する産出弾力性として解釈できます。

事前に書いておかないと、推定結果を見てから「結果に合うように理由を作る」ことが起きやすくなります。1〜2行のメモで十分です。

[7] 単回帰を実行する

statsmodels で単回帰を実行します。formula.api を使うと R のような式で書けます。

import statsmodels.formula.api as smf

model = smf.ols("np.log(gdp_per_worker) ~ np.log(capital_per_worker)", data=df).fit()
print(model.summary())

結果を見るときは、次の順で確認します。

  • 散布図と整合的か（傾きの符号、傾きの大きさのオーダー）
  • 係数の符号は事前の予想と一致するか
  • 一致しないとき、欠落変数や逆因果が考えられるか
  • 解釈を言葉で書けるか（「資本/労働が 1% 増えると GDP/労働が β% 増える」と言えるか）
重要結果は機械的に読まない

\(R^2\) や p 値だけで判断しないこと。卒論では、係数の符号と大きさが事前の経済的な見立てと整合しているか、整合しない場合は何が起きていそうかを必ず議論します。次回の重回帰でも考え方は同じです。
