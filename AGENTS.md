# AGENTS

このリポジトリで作業するときの project philosophy は `contribute/PROJECT.md` を参照すること。

## Non-negotiables

- 教材の対象は、卒論準備中の学部4年生である
- ゴールは、各手法の完全習得ではなく、卒論に必要な分析作法を身につけることである
- 講義ノートは `notes/`、学生向け実行コードは `scripts/` に分ける
- 1回30分のショートセッションを前提に、説明は短く、到達点は絞る
- 理論の深掘りより、データ管理、図表確認、記述統計、再現可能性などの実務を優先する
- データの取得元情報を記録する前提を崩さない
- 元データから分析用データへの加工コードを残す前提を崩さない
- 記述統計量を確認し、先に散布図などの図を描いてから推定に進む流れを重視する
- AI 利用時は、バージョン管理を前提にし、生データや制約付きデータを外部に送らない
- 生データの保存場所は repo 内に決め打ちしない。絶対パスの埋め込みは避ける
- 変更は小さく行い、人間が後でレビューしやすい形を優先する
- 過剰な抽象化や大規模なフレームワーク導入はしない
- `notes/` 内の図・表にはキャプションをつける（Quarto の `: キャプション {#tbl-label}` / `#| fig-cap:` 形式）

## docs ディレクトリの扱い

- `docs/` への変更は必ず単独でコミットする。ソースファイルの変更と同一コミットに混ぜない
- `docs/` をコミットする前に必ず `quarto render` でサイトをビルドし直す
- ビルド中にエラーが1つでも発生した場合は `docs/` をコミットしない
